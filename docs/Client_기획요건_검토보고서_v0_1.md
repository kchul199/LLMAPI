# [검토보고서] Client 기획서/기능요건 개선 검토 (v0.1)

작성일: 2026-04-12

---

## 1. 검토 목적

Client 서버 기획서(v0.2)와 기능요건서(v0.2)를 재검토하여, 실제 개발 착수 전에 누락/리스크 항목을 보완한다.

---

## 2. 검토 결과 요약

### 2.1 강점

- 동기/비동기 처리 경계를 명확히 정의함
- 요청/결과/상태이력 분리 저장 전략이 명확함
- 멱등성, 표준 오류코드, trace_id 정책이 포함됨

### 2.2 보완 필요 항목

1. 보안 요구사항의 구체화
- 서버 간 인증 방식(API Key 또는 mTLS) 선택 기준 필요

2. 데이터 보관 정책의 운영 정의
- 보관기간, 아카이브 위치, 삭제 승인 절차 명시 필요

3. 재처리 정책
- 실패건 재처리 횟수 제한 및 중복 재처리 방지 규칙 필요

4. 운영 SLA
- 동기/비동기별 목표 지연시간과 알람 임계치 수치화 필요

---

## 3. 이번에 문서에 추가 반영한 항목

### 3.1 기획서 반영

- 데이터 거버넌스(보관정책/권한 범위) 추가
- 상태이력 필수화 및 멱등성 필수화
- 장애코드 표준 분류와 재처리 API 후속 항목 추가
- 감사 로그 필드 요구사항 추가

### 3.2 기능요건서 반영

- FR-012~FR-015, FR-019를 Must로 상향
- FR-016, FR-018, FR-020을 운영 고도화 항목으로 추가
- 오류코드 표준 테이블 추가
- 상태 전이 규칙 및 종결 상태 제약 추가

### 3.3 DB 설계서 반영

- 3테이블 분리 설계(request/result/status_history)
- 멱등 유니크 키 추가
- 운영 조회용 인덱스 전략 추가
- 보관/아카이브/보안 운영 가이드 추가
