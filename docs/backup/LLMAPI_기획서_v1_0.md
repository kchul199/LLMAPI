# [기획서] 고객-상담원 대화 기반 NLP 시스템 구축안 (v1.0)

## 1. 프로젝트 개요

*   **목적:** 고객과 상담원 간의 텍스트 대화 데이터를 분석하여 **분류(Category), 요약(Summary), 감정 분석(Sentiment)**을 자동화하고 시스템화하여 업무 효율을 극대화합니다.
*   **배경:** 기존의 수동 분석 방식은 시간과 비용이 많이 소요되며 분석가에 따라 주관적인 결과가 나올 수 있습니다. 이를 최신 대형언어모델(LLM)로 자동화하여 빠르고 객관적인 지표를 확보하고자 합니다.
*   **핵심 전략:**
    *   **신속한 도입:** 별도의 학습(Fine-tuning) 없이 프롬프트 기술만 활용하여 즉시 상용화 수준의 품질을 확보합니다.
    *   **유연한 확장:** 향후 분석 항목이 추가되더라도 시스템 변경 없이 대응 가능한 구조를 지향합니다.
    *   **안정적 운영:** 고성능 GPU(H100) 자원을 체계적으로 배분하고 미들웨어를 통해 과부하를 방지합니다.

## 2. 역할 분담 (R&R)

성공적인 시스템 구축을 위해 클라이언트와 LLM 서버 간의 역할을 다음과 같이 명확히 정의합니다.

| 구분 | 역할 내용 | 비고 |
| :--- | :--- | :--- |
| **클라이언트** (사내 시스템) | **개인정보 마스킹 처리 (PII Masking)**, 분석 요청 전달, 결과 수신 및 표출 | 보안 데이터 처리 전담 |
| **LLM 미들웨어** (LLMAPI) | 요청 큐잉(Queueing), 프롬프트 조립, LLM 추론 요청, 결과 검증 및 반환 | 분석 로직 및 부하 관리 전담 |

> [!IMPORTANT]
> **보안 가이드**: 클라이언트는 상담 텍스트 송신 전 반드시 이름, 전화번호, 주소 등 개인 식별 정보를 마스킹 처리하여 전달해야 합니다. LLMAPI 서버는 비식별화된 데이터를 전제로 분석을 수행합니다.

## 3. 인프라 및 GPU 할당 계획

보유 중인 H100 16개 자원 중 5개를 전략적으로 배치하여 개발부터 운영까지 안정적인 환경을 구축합니다.

| 환경 | GPU 할당 | 물리 배포 | 기대 효과 |
| :--- | :--- | :--- | :--- |
| **개발 (Dev)** | **1장** | Server A | 신규 기능 및 프롬프트 테스트, 코드 개발 |
| **검수 (QA)** | **2장** | Server A | 상용 환경과 동일한 부하 테스트 및 품질 검증 |
| **상용 (Prod)** | **2장** | Server B | 실제 서비스 운영 (물리적 격리를 통한 안정성 확보) |

## 4. 시스템 아키텍처 개요

전체 시스템은 **클라이언트 - 미들웨어(FastAPI/Redis) - 추론 엔진(vLLM)**의 3단계 구조로 구성됩니다.

1.  **입력 단계:** 클라이언트가 마스킹된 텍스트를 API로 전송합니다.
2.  **대기 단계:** 요청이 급증할 경우 Redis 큐에 안전하게 적재되어 순차적으로 처리됩니다. (서버 다운 방지)
3.  **추론 단계:** vLLM 엔진이 2장의 GPU를 하나처럼 활용(TP=2)하여 고속으로 텍스트를 분석합니다.
4.  **반환 단계:** 분석된 결과(JSON)를 클라이언트에 전달하고 로그를 저장합니다.

## 5. 단계별 구축 로드맵 (Roadmap)

*   **Phase 1. 기초 다지기 (개발 환경)**
    *   API 서버 기본 구조 및 메시지 큐 연동.
    *   기본 분석 기능을 위한 프롬프트 템플릿 설계.
*   **Phase 2. 품질 굳히기 (검수 환경)**
    *   다량의 데이터를 활용한 성능(TPS) 측정 및 지연 시간(Latency) 최적화.
    *   다양한 상담 케이스에 대한 프롬프트 미세 조정.
*   **Phase 3. 운영 시작 (상용 환경)**
    *   실제 서비스 배포 및 상시 모니터링 체계 구축.
    *   비즈니스 요구에 따른 분석 항목 지속 확장.

## 6. 기대 효과

1.  **품질 표준화:** 객관적인 AI 분석을 통해 상담 품질 관리의 공정성을 확보합니다.
2.  **운영 효율화:** 상담 후 처리 시간을 단축하여 상담원의 전체 업무 효율을 20% 이상 향상시킬 것으로 기대합니다.
3.  **데이터 자산화:** 정형화된 분석 데이터를 바탕으로 고객 경험 정보를 정량적으로 관리할 수 있습니다.
