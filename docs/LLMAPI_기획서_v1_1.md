# [기획서] 고성능 상담 분석 LLM 시스템 구축안 (v1.1)

## 1. 프로젝트 개요

*   **목적:** 고객-상담원 대화 데이터를 분석하여 분류, 요약, 감정 분석을 자동화하고, **실제 상용 서비스 수준의 안정성과 운영 효율성**을 갖춘 AI 오케스트레이션 플랫폼을 구축합니다.
*   **핵심 가치 (4대 필러):**
    1.  **안정성 (Stability):** 메인 엔진 장애 시 백업 모델로 즉시 전환되는 Fail-over 체계 구축.
    2.  **유연성 (Flexibility):** 코드 변경 없이 프롬프트 및 모델 설정을 실시간 조절 가능한 구조.
    3.  **운영성 (Operational Efficiency):** 전 과정의 가시성 확보를 위한 구조화된 로깅 및 상태 모니터링.
    4.  **확장성 (Scalability):** Redis 기반 비동기 큐를 통한 대규모 트래픽 분산 처리.

## 2. 강화된 역할 분담 (R&R)

| 구분 | 역할 내용 | 비고 |
| :--- | :--- | :--- |
| **클라이언트** | **비식별화(PII Masking)**, 분석 요청, 결과 소비 | 데이터 보안 책임 |
| **LLM 미들웨어** | **장애 대응(Fail-over)**, 프롬프트 최적화, 비동기 큐잉 | 시스템 가용성 책임 |

## 3. 고도화된 아키텍처 전략

### 3.1. 장애 극복(Fail-over) 및 재시도
- **자동 재시도:** 네트워크 일시 오류 시 `tenacity` 기반의 지수 백오프(Exponential Backoff) 재시도 수행.
- **백업 전환:** 상용 모델(70B) 응답 불가 시 로컬 경량 모델(llama3.2:1b 등)로 즉시 전환하여 서비스 연속성 보장.

### 3.2. 운영 가시성 확보
- **구조화된 로깅:** 모든 로그를 JSON 포맷으로 생성하여 대규모 로그 분석 도구와 즉시 연동 가능.
- **상태 확인(Health Check):** 서버 및 LLM 추론 엔진의 건강 상태를 실시간 모니터링하는 API 제공.

### 3.3. 유연한 프롬프트 관리
- **외부 설정화:** 프롬프트를 YAML 파일로 분리 관리하여 배포 없이 프롬프트 고도화 가능.
- **버전 타겟팅:** 요청별로 특정 버전의 프롬프트를 지정하여 호출하는 기능 지원.

## 4. 단계별 구축 로드맵 (Refined)

*   **Phase 1. 기초 및 고도화 (현재)**
    *   API 뼈대 및 로컬 LLM 연동.
    *   **운영성/안정성 코드 반영 (Retry, Fallback, Logging).**
*   **Phase 2. 통합 검증 및 튜닝**
    *   상용 vLLM 서버 연동 및 부하 테스트.
    *   7가지 감정 분석 지표 품질 검증 및 프롬프트 튜닝.
*   **Phase 3. 상용 배포 및 고도화**
    *   Redis 비동기 큐 기반 Scale-out 환경 구축.
    *   프로덕션 모니터링 대시보드(Grafana) 구축.

## 5. 기대 효과 (개선)

1.  **무중단 서비스:** 인프라 장애 시에도 AI 분석 기능이 유지되어 비즈니스 중단 최소화.
2.  **데이터 기반 운영:** 정밀한 로깅을 통해 분석 품질과 비용을 실시간으로 추적 및 최적화.
3.  **신속한 요구 대응:** 프롬프트 관리 효율화를 통해 비즈니스 로직 변경 사항을 즉각 적용.
